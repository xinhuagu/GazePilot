# Training Playbook: From Screenshots to Trained Model

## Complete Workflow

```
gazefy collect  →  gazefy auto-label  →  gazefy prep  →  gazefy train
   (screenshots)     (OmniParser + VLM)    (train/val split)     (YOLO → pack)
```

**No manual annotation required.** OmniParser V2 auto-labels all UI elements. VLM reviews edge cases.

## Step 1: Collect Screenshots

```bash
# Capture from a specific window
gazefy collect --window "Citrix" --pack-name my_erp --interval-ms 500

# Or capture a manual region
gazefy collect --region 100,50,1600,900 --pack-name my_erp --interval-ms 500

# Capture exactly 100 frames
gazefy collect --window "Citrix" --pack-name my_erp --max-frames 100

# Capture for 5 minutes
gazefy collect --window "Citrix" --pack-name my_erp --duration 300
```

Output:
```
datasets/my_erp/session_XXXXXX/
├── images/
│   ├── frame_000000.png
│   ├── frame_000001.png
│   └── ...
├── actions.jsonl
└── dataset.yaml
```

**Tips for good coverage:**
- Walk through every menu, dialog, and page of the application
- Capture different data states (empty forms, filled forms, error states)
- Include disabled button states, loading screens, tooltips
- Aim for 100-200 screenshots covering all UI states

## Step 2: Auto-Label with OmniParser

```bash
# Auto-label all screenshots (zero human effort)
gazefy auto-label datasets/my_erp/session_XXXXXX/

# With VLM review for low-confidence detections (recommended for VDI)
gazefy auto-label datasets/my_erp/session_XXXXXX/ --vlm-review

# Specify confidence threshold for VLM review
gazefy auto-label datasets/my_erp/session_XXXXXX/ --vlm-review --vlm-threshold 0.5
```

**What happens under the hood:**
```
Screenshot
    ↓
OmniParser V2 YOLO → detect all interactable regions (bbox + confidence)
    ↓
Florence-2 → describe each element ("Save button", "Customer ID input field")
    ↓
Class Mapper → map descriptions to YOLO taxonomy (button, input_field, menu_item, ...)
    ↓
YOLO format labels (.txt files, normalized coordinates)
```

**VLM review (--vlm-review flag):**
- Detections with confidence < threshold are sent to Claude Vision in batches
- VLM confirms, corrects, or rejects each detection
- Cost: ~$0.01-0.03 per screenshot, ~$10-30 for 1000 screenshots

Output:
```
datasets/my_erp/session_XXXXXX/
├── images/
│   ├── frame_000000.png
│   └── ...
├── labels/                     ← auto-generated by OmniParser
│   ├── frame_000000.txt
│   └── ...
├── auto_label_report.json      ← labeling stats + confidence distribution
├── actions.jsonl
└── dataset.yaml
```

Each `.txt` file contains YOLO-format annotations:
```
# class_id  x_center  y_center  width  height  (normalized 0-1)
0 0.150 0.020 0.300 0.030    # button
1 0.050 0.020 0.040 0.030    # menu_item
2 0.450 0.180 0.200 0.030    # input_field
```

## Step 3: Prepare Dataset (Train/Val Split)

```bash
gazefy prep datasets/my_erp/session_XXXXXX --split 0.8
```

This splits auto-labeled images+labels into train (80%) and val (20%).

## Step 4: Train

```bash
gazefy train \
  --dataset datasets/my_erp/session_XXXXXX/dataset.yaml \
  --pack-name my_erp \
  --epochs 50 \
  --device mps \
  --window-match "Citrix" "My ERP"
```

This:
1. Trains YOLOv8 on OmniParser auto-labeled data
2. Packages the model into `packs/my_erp/`
3. Creates `packs/my_erp/pack.yaml` with window matching rules

## Step 5: Validate & Iterate

```bash
python scripts/visualize_detections.py \
  --model packs/my_erp/model.pt \
  --images datasets/my_erp/session_XXXXXX/images/val/
```

**Common issues and fixes:**

| Issue | Fix |
|-------|-----|
| Missed small elements (checkboxes, icons) | Increase `--imgsz` to 1280 |
| Too many false positives | Raise confidence threshold in pack.yaml |
| Specific element type always missed | Collect more screenshots, re-run `gazefy auto-label` |
| VDI compression hurting accuracy | Use `--vlm-review` flag, add more VDI screenshots |

**To iterate:**
1. Collect more screenshots: `gazefy collect` (creates a new session)
2. Auto-label: `gazefy auto-label datasets/my_erp/session_NEW/`
3. Merge sessions into one dataset directory
4. Re-run `gazefy prep` and `gazefy train`

Typically 1-2 iterations (vs 3-5 with manual annotation).

## Effort Estimate

| Step | Time | Notes |
|------|------|-------|
| Screenshot collection | 2-3 hours | Systematically cover all UI states |
| OmniParser auto-labeling | ~30-60 min | Fully automated |
| VLM review (optional) | ~10-20 min | Automated API calls, ~$10-30 |
| Training | 1-2 hours | Machine runs, you wait |
| Validation + iteration | 1-2 hours | Auto-refine if needed |
| **Total** | **~½ day** | Down from ~2 days with manual annotation |

## Fallback: Manual Annotation with Label Studio

If OmniParser accuracy is insufficient for your specific application (e.g., highly custom UI controls), you can fall back to manual annotation:

```bash
pip install label-studio
label-studio start
```

1. Import screenshots into Label Studio
2. Draw bounding boxes manually
3. Export in YOLO format
4. Continue with `gazefy prep` and `gazefy train` as normal

See the project DESIGN.md for Label Studio configuration details.
